Help & Support Guidelines, FAQs, and troubleshooting to keep reservations smooth and secure.

Quick Start

Follow these steps to use the reservation system effectively.

  1. Register / Login: Use your staff email and strong password.
  2. Dashboard: View counts, room types, and recent reservations.
  3. Add Reservation: Enter guest details, room type, and dates.
  4. View Reservations: Search by reservation number / guest / room.
  5. Room Details: Confirm rates and available room types.
  6. Logout: Always logout after your shift.

Reservation Guidelines

  • Check-out date must be after check-in date.
  • Nights are calculated automatically from selected dates.
  • Total billing = nights × room rate.
  • Verify contact number before saving.
  • Confirm correct room type for accurate billing.

Security & Best Practices

  • Use strong passwords (8+ chars, upper, lower, number).
  • Do not share accounts — each receptionist should have their own login.
  • Always logout on shared reception computers.
  • Report suspicious errors or unknown logins to admin.

FAQ (Click to Expand)

Check your email and password. If your session expired, login again. If the issue continues, confirm the backend server is running and contact admin.

Confirm correct room type rate and correct check-in/check-out dates. Nights = difference between dates. Total = nights × rate.

Make sure backend is running (npm run dev) and MySQL is started. Also ensure API_BASE in assets/auth.js points to: http://localhost:5050/api

If editing is not enabled in your system version, create a new reservation and inform the admin for cancellation / correction handling.

Troubleshooting

  • Server Error: Ensure backend is running and MySQL credentials are correct in backend/.env.
  • DB Connection Error: Start MySQL service in XAMPP and verify your MySQL password.
  • No data loading: Check API URL in assets/auth.js and confirm you’re logged in.
